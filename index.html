Rule 1
Kin Selection
r·b > c — Hamilton's Rule (1964)
Each agent pays cost c to provide benefit b to a randomly chosen neighbor. Agents are related by r (genetic relatedness: 0 = strangers, 1 = clones). An agent cooperates if the inclusive fitness gain r·b exceeds its personal cost c. When r·b > c, cooperators spread. Agents reproduce proportional to their accumulated payoff, and offspring replace random neighbors — so relatedness emerges naturally as cooperator clusters form. Mutation flips strategy with probability 0.01 each generation.

Rule 2
Direct Reciprocity
w > c/b — w = prob. of next encounter
A population of agents plays the Iterated Prisoner's Dilemma. Strategies: AllC (always cooperate), AllD (always defect), TFT (Tit-for-Tat: cooperate first, then mirror partner's last move), GTFT (Generous TFT: cooperate with prob. 1−c/b after opponent defects). After each round, partners meet again with probability w. When w > c/b, TFT and GTFT can resist invasion by AllD and dominate the population. Fitness is accumulated payoff; agents reproduce and replace neighbors proportionally.

Rule 3
Indirect Reciprocity
q > c/b — q = prob. of knowing reputation
Each agent has a public reputation score (0–1). When agent A is asked to help B, A cooperates if it knows B's reputation (with prob. q) and B's reputation exceeds threshold θ. If A does not know B's reputation, A cooperates with prob. 0.5. Cooperation raises B's reputation; defection lowers it. When q > c/b, agents can identify and selectively help high-reputation cooperators, making cooperation stable. The reputation system acts as a decentralized enforcement mechanism — "I help you, and others will help me because they see me as a cooperator."

Rule 4
Network Reciprocity
b/c > k — k = avg. number of neighbors
The classic Nowak & May (1992) spatial model. Agents occupy a grid and interact only with their k neighbors. Cooperators pay cost c to give benefit b to each neighbor. Defectors give nothing. Each agent accumulates payoff from all neighbor interactions, then the weakest neighbor's strategy is replaced by a copy of the highest-scoring neighbor's strategy. When b/c > k, cooperators form tight clusters where mutual benefit outweighs local exploitation — the exact mechanism absent in well-mixed populations.

Rule 5
Group Selection
b/c > 1 + n/m — n = group size, m = # of groups
The population is divided into m groups, each of size up to n. Within each group, defectors outcompete cooperators (individual selection). But groups with more cooperators accumulate higher total payoff and therefore have higher probability of splitting: the group divides in two, and the smallest other group is eliminated to keep total population size constant. When b/c > 1 + n/m, group-level selection overcomes individual-level defection pressure, and cooperators can dominate the global population.

⬡ UNIFIED SIMULATION — All Five Rules Active

Every agent simultaneously embodies all five mechanisms. Kin selection: agents belong to relatedness clusters — fitness contributions from kin are upweighted by r. Direct reciprocity: each agent carries a strategy (AllC / AllD / TFT / GTFT) and plays iterated games with neighbors, with continuation probability w. Indirect reciprocity: each agent has a public reputation that rises when it cooperates and falls when it defects; agents with low reputation are less likely to receive help. Network reciprocity: agents are fixed on a spatial lattice and only interact with local neighbors — cooperation clusters can form and resist invasion. Group selection: the grid is partitioned into groups; high-cooperation groups periodically expand and replace low-cooperation groups. Toggle each mechanism on/off and tune its strength to observe how they interact.
